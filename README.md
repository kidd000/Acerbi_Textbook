# 文化進化モデルのエージェントベース・シミュレーション
## <a href="https://acerbialberto.com/IBM-cultevo/index.html">Individual-based models of cultural evolution: A step-by-step guide using R, written by Alberto Acerbi,  Alex Mesoudi, &amp;  Marco Smolla</a>

<h2>本書の目的 </h2>
文化進化の分野は、ここ数十年の間に、文化の変化や文化の多様性を進化の枠組みの中で理解し、進化のための道具や概念、方法を用いて理解しようとする、活発な学際的取り組みとして登場しました。進化生物学に根ざしているため、文化的進化の多くはモデルに基づいており、またそれに触発されています。しかし、文化進化に興味を持つ研究者の多くは、心理学、人類学、考古学など、モデルの訓練を受けていないバックグラウンドを持っています。<br><br>
本書の目的は、このギャップを少しでも解消することであり、文化進化の個人ベースのモデル（IBM，エージェントベースモデル，ABMとも呼ばれる）を作成する方法を読者に紹介することである。偏った伝達（バイアスのある伝達）や文化的な突然変異（イノベーション）といった文化進化の基本的なプロセスの非常にシンプルなモデルから、社会的学習の進化、人口統計学的効果、ソーシャルネットワーク分析といったより高度なトピックまで、さまざまなモデルを用意しています。可能な限り、文献にある既存のモデルを再現することで、読者が既存のモデルをよりよく理解できるようにし、さらには読者自身が興味を持っている問題を解決するためにモデルを拡張できるようにしています。

### 文化進化とは何か？
進化論は、典型的には遺伝子の変化に適用されます。ダーウィンは、生物の多様性や複雑性は、非常に単純なプロセスで説明できることを指摘した。すなわち、（1）生物はその特性が異なる、（2）これらの特性は親から子へと継承される、（3）生物の生存と繁殖の可能性を高めるような特性は、時間の経過とともに頻度が増加する傾向にある、と。以上が、ほぼ全てです。ダーウィン以来、生物学者はこの抽象的な考えの詳細を多く明らかにしてきた。遺伝学者は、遺伝可能な「特性」が遺伝子によって決定されることを明らかにし、遺伝子の変異がどこから来るのか（突然変異、組み換えなど）、遺伝がどのように作用するのか（メンデルの法則やDNAなど）を解明しました。選択の詳細を調べ、ある遺伝子が広がり、他の遺伝子が広がらない多くの理由を明らかにしました。また、すべての生物学的変化が淘汰された結果ではなく、集団のボトルネック（遺伝的ドリフト）のようなランダムなプロセスによっても生じることがわかった。
<br><br>
現代の文化進化論は、文化が上記と同様の進化プロセスを構成しているという観察から始まりました。「文化」とは、遺伝的にではなく社会的に個人から個人へと伝達される情報と定義される。これには、俗に言う知識、信念、アイデア、態度、習慣、言葉、価値観などが含まれます。これらはすべて、模倣や言語（話し言葉・書き言葉）など、さまざまな「社会的学習」のメカニズムを介して他者から学ばれます。重要な点は、社会的学習が継承システムであるということです。文化的特性（または文化的形質）は個人によって異なり、個人から個人へと受け継がれ、多くの場合、ある形質は他の形質よりも広まりやすいのです。これは、ダーウィンの洞察を文化に応用したものです。文化進化の研究者たちは、生物学者が生物の多様性や複雑性を説明したように、私たちも同じような進化の概念やツール、手法を使って文化の多様性や複雑性を説明できると考えている。本書のモデルは、文化進化の様々な側面のシミュレーションを作成することで、上記の原理の多くを理解するのに役立ちます。 <br>
重要なのは、文化的進化が遺伝子的進化と同一であると仮定する必要はないということです。多くの細部は異なるでしょう。例えば、DNAは両親からしか得られませんが、アイデアは、先生、インターネット上の見知らぬ人、亡くなった作家の本、あるいは両親など、さまざまなところから得ることができます。文化進化の研究者たちは、これらの詳細を埋めるためにモデルを構築し、研究を行っています。本書の終盤では、生物進化との厳密なアナロジーを超えて、伝達を規制する「ルール」自体が文化的に進化することや、継承以外のプロセスが文化を創造し安定させることなどに焦点を当てたモデルについても検討します。

### どうしてモデルなのか？
形式モデル（formal model）とは、現実を簡略化したもので、数式やコンピュータコードで書かれています。形式的なモデルが有効なのは、現実が複雑だからです。私たちは、種や文化の時間的変化や、生物学的または文化的な多様性の特定のパターンを観察することができますが、特定のパターンや傾向には常に膨大な数の原因があり、そして膨大な数の変数が様々な方法で相互作用しています。形式的なモデルは、この複雑な現実のごく一部を非常に単純化して再現したもので、モデル作成者が重要だと考える要素やプロセスをいくつか含んでいます。モデルは、現実とは異なり、各部分がどのように機能するかをよりよく理解するために、操作したり調べたりすることができます。現実を完全に再現したモデルはありません。複雑で理解できない現実を、複雑で理解できないモデルに置き換えてしまうことになるからです。むしろ、モデルはそのシンプルさゆえに有用なのです。
<br><br>
社会科学の分野では、経済学などの例外を除き、形式的なモデル化はほとんど行われていません。社会科学者は、人間の文化のように非常に複雑なものについて、非常に単純なモデルが何か役に立つことを教えてくれるとは思えない、と懐疑的になる傾向があります。しかし、生物学から得られる明確な教訓は、まさにこのような状況でモデルが非常に役立つということです。生物学者は、自然界でも同じように膨大な複雑さに直面しています。それにもかかわらず、モデルは役に立つのです。20世紀初頭の集団遺伝学のモデルは、遺伝学の新しい発見とダーウィンの進化論を両立させるのに役立ちました。生態学モデルは、捕食者と被食者のサイクルなど、種間の相互作用を理解するのに役立ちました。これらのモデルは非常に単純化されています。集団遺伝学のモデルは通常、無限に大きな集団やランダムな交配など、とんでもない仮定をしています。しかし、これらのモデルは、複雑なシステムの各部分を正確に特定し、現実の理解を深めるという点で有用です。
<br><br>
別の見方をすれば、社会科学者はみなモデルを使っていますが、形式的なモデルを使っているのは一部の人だけです。社会科学の理論のほとんどは、言葉（自然言語）で書かれたモデルです。問題は、言葉は不正確な場合があり、言葉によるモデルにはあらゆる種類の隠れた、あるいは明記されていない仮定が含まれていることです。形式的なモデル化の利点は、提案するすべての要素とプロセスを正確に指定し、すべての仮定を明示することを余儀なくされることです。数学やコードは曖昧さを許さないので、すべてを語らなければなりません。
<br><br>
モデルは、私たちの理論の結果を理解するのにも役立ちます。社会システムは、他の多くのシステムと同様に、通常、複数の異なる相互作用する力の影響下にあります。これらの力の影響は、単独では簡単に予測できます。しかし、いくつかの力が相互に作用すると、結果的にダイナミクスはすぐに自明ではなくなります。これが、システムを「複雑系」と定義する基本的な考え方です。言葉による説明では、相互作用の影響を理解するのは我々の洞察力に委ねられます。形式的なモデルでは、これらの力を持つシステムを設定し、その相互作用のダイナミクスを観察することができます。

### どうして"エージェントベース"モデルなのか
形式的なモデルにはいくつかの異なるタイプがあります。モデルの中には、個体を明示的にモデル化することなく、形質の全体的な頻度やその他の記述的統計を追跡し、集団レベルでシステムの挙動を記述するものがあります。例えば、ある文化的特徴Aのt時点での頻度はt-1時点での頻度に依存すると規定するモデルがあります。時間ステップごとに2倍になるようなモデルです。その代わりに、形質を持つ個々の実体を明示的にモデル化して、個体レベルでのシステムの挙動を記述するモデルもあります。さきほどと同じ問題を想像してみてください。ここでは、N個の個体からなる集団において、各個体が毎回ランダムな数の他の個体を観察し、そのうちの少なくとも1つが形質Aを持っていれば、その形質をコピーする、と規定します。
<br><br>
もう一つの違いは、解析的に扱いやすいモデルとそうでないモデルです。解析的に扱いやすいモデルとは、特定の答え（例えば平衡点）を求めることができる、方程式の集合からなる数学的モデルです。先ほどの集団レベルのモデルは、このタイプのモデルです。これらのモデルの大きな利点は、広い範囲のパラメータに対するシステムのダイナミクスの洞察や、特定の問題に対する正確な結果を提供できることです。しかし、この方法では、研究対象となるダイナミクスがかなり単純であることが必要です。2つ目のモデルでは、単一の個体の行動を記述するのに必要な方程式系を書いて解析的に解くことは、より困難（あるいは不可能）でしょう。
<br><br>
しばしば、個人レベルでの行動を記述したい、または記述する必要がある場合、例えば、個人の特性が異なっていたり、学習や適応を行ったり、社会的ネットワークに組み込まれていたりする場合、方程式系を書こうとすることは最良の戦略ではないかもしれません。むしろ、コードを書いてコンピュータプログラムを実行させる必要があります。これがエージェントベースモデルです。これらのモデルは、個人レベル（つまり、個人の特性と、それらの個人が相互に作用したり、時間とともに変化したりするいくつかのルールを規定する）と、シミュレーション（つまり、分析的に解決するのではなく、コンピュータプログラムを通じてシミュレーションする）の両方からなります。
<br><br>
シミュレーションは、解析的なモデルよりも柔軟性があります。また、その構造から、特に数学の訓練を受けていない人でも直感的に理解できることが多い。しかし、その欠点にも注意することが重要です。例えば、一般化はできないことが多く、主張はシミュレーションされたパラメータ（またはそのセット）に対してのみ有効です。また、シミュレーションの柔軟性の高さから、すぐにモデルが複雑になり、モデル内部で何が起こっているのかを理解するのが難しくなることもあります。だからこそ、私たちのエージェントベースモデルは理解できるほどシンプルで、文化的進化のモデル化への入り口となることを願っています。

## 本書の構成
本書は以下のように構成されています。まず、Robert BoydとPeter Richersonの<a href="https://acerbialberto.com/IBM-cultevo/references.html#ref-boyd_culture_1985">『Culture and the Evolutionary Process』</a>やLuigi-Luca Cavalli-SforzaとMarc Feldmanの<a href="https://acerbialberto.com/IBM-cultevo/references.html#ref-cavalli-sforza_cultural_1981">『Cultural Transmission and Evolution』</a>など、文化的進化の基礎となった書籍に記載されている、今では定番となった数学的モデルや集団レベルのモデルのいくつかを、エージェントベースモデル・バージョンとして紹介しています。これらのモデルは、文化的進化を記述するモデルを構築するためのいくつかの基本的なツールを提供しています。続くいくつかの章では、例えば、多くの異なる形質を持つ文化的ダイナミクスのシミュレーション（第7章）など、エージェントベースモデルでのみ可能な側面を展開しています。
<br>
続いて、"Advanced topics"に移ります。これらの章では、文化的進化に関する最近の研究を扱っており、異なる視点や文化的進化のモデリングでは一般的ではない分析（例えば、第14章のネットワーク分析）に関するものも含まれています。
